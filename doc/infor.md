1. 개요 (PoC 정의)

'패션 AI 가상 피팅 에이전트' 개발의 첫 단계인 PoC(기술 검증)의 목표, 범위, 아키텍처 및 상세 계획을 정의합니다. 현재 팀이 보유한 '2D to 3D 자동 변환 기술'을 핵심 엔진으로 활용하되, 이를 단순 기능 호출 방식이 아닌 지능형 에이전트(Intelligent Agent)가 주도적으로 전체 프로세스를 오케스트레이션(Orchestration)하는 방식의 기술적 타당성을 검증하는 것을 핵심 목표로 합니다.

PoC에서 구현할 에이전트는 사용자의 복합적인 의도를 인식(Perception)하고, 최적의 작업 계획을 판단(Judgment)하며, 필요한 기술 모듈(도구)을 자율적으로 사용하여 결과를 행동(Action)으로 옮기는 지능적인 문제 해결 프로세스를 따릅니다.

다. 작동원리 (Operating Principle)
    본 에이전트는 단순한 LLM 호출(LLM Workflow)이나 정보 검색(RAG)을 넘어, 여러 에이전트가 자율적으로 협력하는 Agentic AI 패러다임을 지향합니다.



[그림 4]. AI 상호작용 패러다임의 진화 (Evolution of AI Interaction Paradigms)

    
   ▮ Agentic AI 패러다임은 기존 방식에 비해 '역할 분담을 통한 팀플레이'가 가능하다는 압도적인 장점을 가집니다. 본 PoC가 지향하는 ①Agentic AI는 총괄 감독(Agent 1)이 “전략을 수립”하고, ②작업 지시 전문가(Agent 2)가 이를 “구체화”하며, 각 ③도구(Extensions)가 실제 작업을 수행하는 전문가 팀처럼 움직입니다. 이를 통해 단일 에이전트로는 해결하기 어려운 복잡한 문제를 훨씬 더 안정적으로 해결할 수 있습니다.

       본 PoC에서는 바로 이 '팀플레이' 방식의 Agentic AI를 아래와 같이 
       '인식-판단-행동'이라는 3단계의 구체적인 작동 원리로 구현합니다.

       1. 인식 (Perception): 사용자가 UI를 통해 "이 옷을 입혀줘"라는 텍스트와 함께 2D 의류 이미지를 입력합니다. 멀티모달 모델(VLM)은 이 입력을 분석하여 '사용자는 제공된 옷 이미지로 가상 피팅을 원한다'는 최종 목표를 인지합니다.
       2. 판단 (Judgment / Orchestration):
          ▫ 종합 감독 에이전트(Agent 1)는 사용자의 목표를 달성하기 위한 큰 그림의 작업 계획(예: 의류 이미지 분석 → 3D 모델 생성 → 렌더링)을 수립합니다.
          ▫ 작업 지시 전문가 에이전트(Agent 2)는 이 계획을 받아, 각 단계를 실행할 구체적인 도구(API) 호출 순서와 파라미터를 담은 JSON 형식의 실행 계획을 생성합니다. 이 과정에서 필요시 데이터 스토어를 참조(RAG)하여 더 정확한 계획을 세웁니다.
       3. 행동 (Action):
          ▫ 생성된 실행 계획에 따라 Agent Runtime은 '2D 처리 모듈', '3D 변환 모듈' 등 필요한 Extensions(도구)를 순차적으로 호출합니다.
          ▫ 모든 작업이 완료되면, 최종 생성된 3D 가상 피팅 결과물이 사용자 UI의 3D 뷰어에 시각화되어 전달됩니다.

에이전트가 자신이 생성한 결과물이 올바른지 스스로 검증하고 평가

5. 예외 처리 및 복구
작업 수행 중 오류가 발생했을 때, 이를 감지하고 시스템이 멈추지 않도록 대처

6. 병렬화
여러 작업을 동시에 실행하여 전체 처리 시간을 단축
아바타 모델 렌더링과 추천 의상 목록 조회를 동시에 실행하여 사용자 대기 시간 감소
7. 계획
사용자 프롬프트를 분석하여 최적의 작업 계획을 능동적으로 수립
"이 셔츠와 어울리는 바지를 추천하고 입혀줘" 요청에 대해 [스타일 분석 → 바지 검색 → 가상 피팅]의 다단계 계획을 자동 수립
8. 다중 에이전트 
   협업
각기 다른 전문성을 가진 에이전트들이 협력하여 복잡한 문제를 해결
'스타일 추천 에이전트'와 '재고 확인 에이전트'가 협력하여 '지금 바로 구매 가능한 어울리는 상의'를 추천
9. 메모리 관리
사용자의 과거 대화나 선호도를 기억하여 개인화된 경험을 제공
과거에 "스트리트 패션"을 선호했던 사용자에게는 오버사이즈 후드티를 우선적으로 추천
10. 지식 검색 (RAG)
외부 데이터베이스의 최신 정보를 검색하여 답변이나 판단의 정확도를 높임
최신 패션 트렌드나 쇼핑몰 재고 데이터를 실시간으로 검색하여 더 정확한 의상 추천
11. 가드레일/안전
부적절한 요청을 차단하고 시스템이 안전한 범위 내에서 작동하도록 보장
사용자가 부적절하거나 관련 없는 이미지를 업로드 시, 이를 감지하고 정중하게 요청을 거절
12. 평가 및 모니터링
시스템의 성능(응답 시간, 성공률 등)을 지속적으로 측정하고 추적
가상 피팅 평균 소요 시간을 로깅하고, 목표치(30초)를 초과할 경우 개발팀에 알림 전송
13. 자원 인식 최적화
작업의 복잡도에 따라 비용 효율적인 자원(모델)을 동적으로 선택
단순 색상 변경은 경량 모델(SLM)로, 복잡한 3D 생성은 고성능 모델(LLM)로 처리하여 비용 최적화
14. 인간 참여 루프
AI의 판단이 불확실할 때, 사람에게 개입을 요청하여 결정의 안정성을 높임
AI가 추천한 스타일 코디가 애매할 경우, "A와 B 중 어떤 스타일이 더 마음에 드시나요?"라고 사용자에게 되물음
15. 에이전트 간 통신 (A2A)
표준화된 프로토콜을 사용해 에이전트들이 원활하게 데이터를 교환하고 협업
스타일 에이전트가 재고 에이전트에게 표준 A2A 포맷으로 상품 코드를 보내 재고 유무를 실시간으로 확인
16. 목표 설정 및 
    모니터링
사용자가 설정한 장기적인 목표나 제약 조건을 대화 내내 추적하고 관리
"30만원 예산으로 하객룩 완성하기" 목표를 설정하면, 대화가 이어지는 동안 모든 추천 아이템의 총합이 예산을 넘지 않도록 관리
17. 우선순위 지정
여러 작업 요청이 동시에 들어왔을 때, 중요도에 따라 처리 순서를 동적으로 관리
유료 구독자의 가상 피팅 요청을 일반 사용자의 요청보다 우선적으로 처리하여 더 나은 서비스 경험 제공
18. 추론 기법 
    (CoT 등)
논리적 사고 과정을 통해 복잡한 질문에 대한 합리적인 답변을 도출
"이 재킷이 여름에 더울까?" 질문에 대해 [재질 분석 → 계절 특성 고려 → 결론 도출]의 단계적 추론 과정을 거쳐 답변
19. 학습과 적응
사용자의 피드백과 상호작용 경험을 통해 점진적으로 성능을 개선
사용자가 특정 브랜드의 옷을 반복적으로 '좋아요' 표시하면, 해당 브랜드의 추천 가중치를 높여 개인화 정확도 향상
20. 탐험과 발견
새로운 지식이나 트렌드를 능동적으로 탐색하여 시스템의 지식 기반을 확장
주기적으로 패션 커뮤니티와 SNS를 크롤링하여 '발레코어 룩' 같은 신조어를 학습하고 추천에 활용
21. 모델 컨텍스트 
    프로토콜 (MCP)
확장된 표준 인터페이스를 통해 새로운 외부 도구나 에이전트와 쉽게 연동
제3의 '신발 사이즈 추천' 전문 에이전트를 추가할 때, MCP를 통해 별도 개발 없이 즉시 데이터 연동


 3. Pilot 2단계 적용 예정: 서비스 고도화 및 자율 운영 체계 구축

상세 설명
기대효과
1. 패턴 융합 및 시너지 극대화
개별적으로 작동하던 패턴들을 복합적으로 연계하여, 1+1=3 이상의 시너지를 창출. 
(예: 다중 에이전트 협업 + 우선순위 지정 + 자원 최적화)
(동적 에이전트 오케스트레이션) VIP 사용자의 복잡한 코디 요청이 들어오면, 시스템은 이를 고부가가치 작업으로 인지합니다. (우선순위 지정) 다른 일반 요청보다 먼저 처리하도록 순서를 조정하고, (다중 에이전트 협업) '스타일', '재고', '렌더링' 에이전트를 동시에 할당하며, (자원 최적화) 가장 성능이 좋은 GPU와 LLM 리소스를 이 작업에 집중 배분하여 최고의 서비스 경험을 제공합니다.
2. 자율적 목표 추론 및 장기 계획 수행
사용자의 단편적인 요청을 넘어, 암시적인 장기 목표를 스스로 추론하고, 여러 세션에 걸쳐 목표 달성을 위한 계획을 능동적으로 수행. 
(예: 목표 설정 + 계획 + 메모리 관리)
('웨딩 게스트 룩' 컨설팅) 사용자가 "다음 달 친구 결혼식에 입고 갈 옷 좀 찾아줘"라고 말하면, 에이전트는 '결혼식 하객 룩 완성'이라는 (목표 설정)을 합니다. 이후 며칠에 걸쳐 "오늘은 드레스 코드부터 정해볼까요?", "어제 고른 원피스와 어울리는 구두를 찾아봤어요" 와 같이 (계획)에 따라 대화를 먼저 이끌어 나갑니다. 이 모든 과정은 (메모리 관리)를 통해 여러 세션에 걸쳐 기억됩니다.
3. 선제적 상호작용 및 개인화 심화
사용자가 요청하기 전에 먼저 필요를 예측하여 유용한 정보를 제안하는 '프로액티브(Proactive)' 에이전트로 진화. 
(예: 학습과 적응 + 탐험과 발견)
(개인화 스타일리스트) 에이전트는 사용자가 이전에 구매했던 청바지 스타일을 (학습과 적응)을 통해 기억합니다. 이후 주기적으로 신상품 데이터를 (탐험과 발견)하다가, 해당 청바지와 완벽하게 어울리는 스타일의 신상 재킷이 입고되면 사용자에게 먼저 푸시 알림을 보냅니다. "고객님이 보유하신 OOO 청바지와 잘 어울리는 신상 재킷이 입고되었습니다. 가상으로 입어보시겠어요?"
4. 엔터프라이즈급 최적화 및 안정성 확보
대규모 사용자를 감당할 수 있도록 비용, 성능, 안정성을 극한으로 최적화하고, 시스템 스스로 이상 상황에 대처하는 회복탄력성 강화. 
(예: 평가 및 모니터링 + 예외 처리 및 복구)
(자율 복구 시스템) 시스템은 실시간으로 API 응답 시간과 에러율을 (평가 및 모니터링) 합니다. 특정 3D 변환 모듈의 응답이 느려지는 것을 감지하면, (예외 처리 및 복구) 로직에 따라 자동으로 트래픽을 예비 서버로 우회시키고, 담당 개발자에게 장애 경고 알림을 보냅니다. 이 모든 과정이 사람의 개입 없이 자동으로 이루어져 서비스 중단을 방지합니다.
